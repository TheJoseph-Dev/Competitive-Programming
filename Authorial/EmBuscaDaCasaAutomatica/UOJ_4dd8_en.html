In Quest for the Automatic House

By José Vitor V. Martos, UNOESTE Brazil

Timelimit: 1

Minecraft is a sandbox voxel-based game in which the player can interact with the world in many ways.

Viniccius13 is one of the best players in the history of the game. After many farms and adventures, he decided that the time had come to complete his final quest: The Automatic House!

To do this, Viniccius13 chose 5 different types of blocks that will constitute the house. He already has an idea of ​​how it will be and has selected a space to build it (no desert or mountain was excavated in the process).

Although Viniccius13 is an excellent player, he still has difficulty with decorations. Therefore, it may be necessary to make changes to the blocks in any coordinate (x, y, z). Aiming to start building, Viniccius13 wants to know how many blocks will be used in each part of the house, in order to prepare and gather all the resources necessary to complete his mission.

 

Input

The input consists of a single test case. The first line contains an integer N (1 ≤ N ≤ 105), representing the number of operations. The next N lines contain the operations, which will take one of the following forms:

    1 X Y Z B: "Place a block of type B at position (x, y, z)" (1 ≤ x, y, z ≤ 100, 1 ≤ B ≤ 5)

    2 X Y Z U V W: "How many blocks of each type are needed to build in the cuboidal space defined by (x, y, z) up to (u, v, w)" (1 ≤ x, y, z, u, v, w ≤ 100, x ≤ u, y ≤ v, z ≤ w)

Assume that at the beginning, no blocks are placed.

Output

For each type 2 operation, print 5 integers representing the number of blocks of types 1, 2, 3, 4 and 5, in that order, contained within the defined volume.
